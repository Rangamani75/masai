Client & Server communication

How a web request works

  1. The user, through a web browser, enters a URL into the address bar or clicks a link.
  2. The browser recognizes the "http" or "https" part of the URL, indicating the Hypertext Transfer Protocol should be used.
  3. The browser needs to find the server's IP address, so it uses a DNS (Domain Name System) server to resolve the domain name (e.g., google.com) to its corresponding IP address.
  4. The browser uses TCP/IP to establish a connection with the server at the identified IP address.
  5. The browser sends an HTTP request to the server over the established TCP connection.
  6. The server receives the request and processes it to determine what to send back.
  7. The server receives the request and processes it to determine what to send back.
  8. The browser receives the HTTP response and renders the content, displaying it to the user.

Fundamental protocols of the internet:

  • HTTP (Hypertext Transfer Protocol): The protocol used for transferring web pages on the internet.
  • HTTPS (HTTP Secure): A secure version of HTTP that encrypts data between the client and server.
  • TCP/IP (Transmission Control Protocol/Internet Protocol): The fundamental communication protocols of the internet.
  • DNS (Domain Name System): The system that translates domain names into IP addresses.
  • FTP (File Transfer Protocol): A protocol used for transferring files between computers on a network.
  • SMTP (Simple Mail Transfer Protocol): The protocol used for sending emails.

Client vs Server

function Client Server
Requests resources Initiates requests for resources or services. Responds to requests from clients and provides resources or services.
Processes data locally Typically runs on a user's device (e.g., web browser, mobile app). Runs on a remote machine (e.g., web server, database server).
Interacts with the user Examples: Web browsers, email clients, mobile apps. Examples: Web servers, database servers, application servers.
Handles user input Requests data or services from the server. Processes requests and sends back the requested data or services.

Static vs Dynamic Features

Feature Static Dynamic
Definition Content that does not change or is fixed. Content that can change or is generated dynamically.
Examples HTML pages, images, CSS files. Web applications, user-generated content, databases.
Interactivity Limited interactivity; users cannot modify the content. Highly interactive; users can modify or interact with the content.
Loading time Generally faster loading times as content is pre-built. May have slower loading times due to server-side processing.
Server-side processing No server-side processing required; content is served as-is. Requires server-side processing to generate content based on user input or other factors.
